# Decision Rules

## Decision rule 1: Should we remove this onboarding step?
Remove it only if it teaches nothing, learns nothing useful, or does not improve routing, confidence, or lifecycle performance.

## Decision rule 2: Should we shift toward bigger bets?
Yes, if AI is central to product value and future capability expansion materially changes the opportunity set.

## Decision rule 3: Should AI be allowed to run this experiment?
Only if the guardrails are explicit, the risk is low or bounded, and outcomes can be measured.

## Decision rule 4: Does this project need a PM?
Usually yes if it exceeds a small size threshold, has significant stakeholder complexity, or is controversial.

## Decision rule 5: Does this need a PRD?
Only if the stakes, complexity, or risk justify it. Otherwise use the lightest viable coordination mechanism.

## Decision rule 6: Should we run this controversial growth idea?
Not if it violates a red line. If it is only high sensitivity, require stronger upside and tighter review.

## Decision rule 7: Should we revisit our tooling assumptions?
Yes whenever there is a meaningful model or tooling upgrade.

## Decision rule 8: Is this org losing focus?
Probably, if constraints are increasing and the number of simultaneous strategic bets is not decreasing.
