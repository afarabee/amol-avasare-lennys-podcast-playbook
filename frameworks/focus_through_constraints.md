# Focus Through Constraints

## Classification
Framework, not a runtime skill.

## Why it moved out of /skills
This is a strong prioritization lens, but it behaves more like a strategic narrowing framework than a direct execution skill.

## When to use this framework
- Leadership is facing too many possible directions
- Resources are constrained
- A product needs a sharper wedge
- A team is mistaking optionality for strategy

## Core idea
Constraints can reduce noise and force sharper choices. Narrow focus is often the fastest path to escape velocity.

## Use this framework to assess
- which constraints are most binding
- which paths fit those constraints best
- what should be explicitly deprioritized
- what wedge gives the highest chance of compounding

## Recommended output
Return:
- key constraints
- strategic implications
- recommended wedge
- what to defer
- rationale
- failure mode if focus is not narrowed
