# Continuous Tool Revalidation

## Classification
Framework, not a runtime skill.

## Why it moved out of /skills
This content is a recurring operating discipline, but it is not normally selected as a just-in-time skill unless explicitly requested. It works better as a standing framework.

## When to use this framework
- A team is relying on stale assumptions about AI tools
- A major model or product release changes capability expectations
- Leaders want a structured cadence for re-testing workflows

## Core idea
Capabilities change fast enough that “didn’t work before” is weak evidence. Re-test tools and workflows systematically as the frontier moves.

## Use this framework to assess
- which workflows should be retested
- what prior assumptions should be revisited
- what has changed since the last model release
- what adoption guidance needs updating

## Recommended output
Return:
- workflow tested
- prior assumption
- new findings
- what changed
- recommended adoption level
- next retest date
